# MyanMate 블로그 운영 가이드

공동창업자는 Keystatic에서 글과 이미지를 편집하고, 방문자는 공개된 글만 읽습니다.

## 최초 1회 권한 설정

1. GitHub의 `JunOJu95/MyanMate-public` 저장소를 **Private**으로 유지합니다.
2. 공동창업자의 GitHub 계정을 저장소 Collaborator로 초대하고 **Write** 권한을 부여합니다.
3. Keystatic용 GitHub App이 이 Private 저장소에 접근할 수 있도록 설치 범위를 확인합니다.
4. Vercel의 Git 연동도 이 Private 저장소에 접근할 수 있는지 확인합니다.
5. Vercel Production 환경변수에 `.env.example`의 Keystatic 변수 4개를 설정합니다.

Keystatic의 GitHub 저장 방식은 블로그 컬렉션만 따로 권한을 나누지 않습니다. 편집자는 저장소 전체에 Write 권한을 갖게 되므로 공동창업자 계정만 초대하고 GitHub 2단계 인증을 켜는 것을 권장합니다.

## 글 작성

1. 배포 사이트의 `/keystatic/branch/main`에 접속해 GitHub로 로그인합니다.
2. **Blog · 블로그**를 선택하고 새 글을 만듭니다.
3. 다음 필드를 채웁니다.
   - Internal title: 영문 URL용 제목
   - Language: 글 본문 언어 하나
   - Category: Visa, Jobs, Housing, Korean, Daily life 중 하나
   - Headline / Excerpt / Author
   - Published date / Last reviewed
   - Cover image와 이미지 설명(선택)
   - Body: 본문과 본문 이미지
4. 본문 제목은 Heading 2부터 사용합니다. 페이지의 Headline이 이미 Heading 1입니다.
5. 이미지에는 화면을 보지 못하는 독자도 이해할 수 있는 설명을 입력합니다.
6. 처음에는 **Draft (keep private)** 를 켠 채 저장합니다.

대표 이미지는 가로형 16:9, 약 1600×900px의 JPG 또는 WebP를 권장합니다. 업로드가 너무 느리지 않도록 보통 1.5MB 이하로 준비합니다.

## 검토와 발행

1. 사실·날짜·공식 출처 링크를 확인하고 **Last reviewed**를 실제 확인일로 갱신합니다.
2. 정책성 정보는 독자가 최종 요건을 공식 기관에서 다시 확인할 수 있게 씁니다.
3. 첫 메시지나 글에서 여권번호, 외국인등록번호, 금융정보, 서류 사진 등 민감정보를 받지 않습니다.
4. 다음 사업 경계를 확인하고 **Publishing boundary confirmed**를 켭니다.
   - 정보 제공·동행 자문 범위의 글이며, 서류 작성·신청·제출을 대신한다고 표현하지 않았습니다.
   - 특정 매물을 소개·연결하거나 조건 협상·계약 관여를 약속하지 않았습니다.
5. 준비가 끝나면 **Draft (keep private)** 를 끄고 저장합니다. 경계 확인이 없는 공개 글은 빌드가 중단됩니다.
6. GitHub 저장과 Vercel 배포가 끝난 뒤 다음을 확인합니다.
   - `/blog` 목록과 카테고리 필터
   - `/blog/{글 주소}` 상세
   - 홈의 최신 글 영역
   - 대표 이미지와 본문 이미지
   - 모바일에서 미얀마어 줄바꿈과 이미지 폭

저장 후 배포 전까지는 이전 공개 버전이 계속 보일 수 있습니다. Vercel 배포가 완료된 다음 공개 페이지를 확인하세요.

## 초안과 이미지 보호 방식

- 글 원본: `src/content/posts/{slug}.mdoc`
- 비공개 이미지 원본: `src/content/blog-media/{slug}/`
- 공개 빌드용 이미지: `public/images/blog/{slug}/` (자동 생성, Git 미추적)

`npm run dev`는 로컬 미리보기를 위해 공개 글과 초안이 참조하는 이미지를 모두 임시 공개 폴더로 복사합니다. `npm run build`는 Draft가 꺼진 글이 참조하는 이미지만 복사합니다. Keystatic에서 이미지를 새로 저장한 뒤 로컬 페이지에 바로 보이지 않으면 개발 서버를 다시 시작하거나 `npm run blog:assets:all`을 실행합니다.

Private 저장소 전환 전에 실제 초안이나 미공개 이미지를 저장하지 마세요. 저장소를 나중에 Private으로 바꾸더라도 이미 공개된 Git 기록이나 포크의 사본까지 회수되지는 않습니다.

## 문제 해결

- 로그인할 수 없음: GitHub Collaborator 초대 수락, GitHub App 설치 범위, Vercel 환경변수를 순서대로 확인합니다.
- 저장은 됐지만 글이 안 보임: Draft가 켜져 있는지, Vercel 배포가 완료됐는지 확인합니다.
- 빌드에서 이미지 누락 오류: 글에 저장된 이미지 주소와 `src/content/blog-media/{slug}/`의 실제 파일을 확인합니다.
- 글이 빌드되지 않음: 필수 필드, 본문, 날짜, 이미지 설명을 확인합니다. 공개 글의 본문 오류는 조용히 누락되지 않고 빌드를 중단하도록 설정되어 있습니다.
